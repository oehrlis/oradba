# Technical Debt Register - OraDBA v0.24.11

**Generated:** 2026-06-26 **Basis:** consolidated-findings.md (CF-NNN) and the 8 domain reviews
**Target:** v1.0.0 stable

Technical debt = structural/maintainability cost that is not itself an acute defect but raises the
cost of every future change or carries interest if deferred. Acute defects (zero-start arithmetic,
credential exposure) are tracked as findings/risks; this register captures the debt that produced or
surrounds them.

Effort tiers: S (\< 1 day), M (1-3 days), L (3-10 days), XL (\> 10 days). Milestone hint: the
roadmap agent formalizes these - these are suggestions only.

## Suggested remediation sequencing

Foundational items unblock the rest:

1. **First (foundational, low-risk enablers):** TD-13 (lint guard + pin + hook), TD-16 (fix the
    test gate), TD-04 (single interface definition), TD-19 (build-injected versions). These are
    mostly S-effort and protect later work.
2. **Then (blocker defect classes):** TD-05 + TD-06 together (increment guards land with strict
    mode), TD-21 (credential/temp-file discipline), TD-11 + TD-12 (portability and runtime
    validation).

    DECISION-REQUIRED) -\> TD-03 (plugin loader, DECISION-REQUIRED) -\> TD-09 (namespace prefixing).
    TD-07 and TD-08 follow.
4. **Then (coverage and release):** TD-14 + TD-15 (land on the path TD-02 selects), TD-17
    (integrity gates, after TD-16), TD-18 (docs), TD-22 (readiness definition, last - it depends on
    knowing the rest is closed).
5. **Deferrable to v1.1.0:** TD-10 (hot-path performance, except CF-015 sqlplus gating which is a
    v1.0.0 blocker) and TD-20 (supply-chain reproducibility).
