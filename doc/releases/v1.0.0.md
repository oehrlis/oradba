# Release Notes v1.0.0

<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-07-09
**Type:** General Availability (GA)

## Overview

Version 1.0.0 is the first stable release of the OraDBA base environment framework.
It ships after eight release-candidate cycles (rc.1 through rc.8) that progressively
hardened the installer, extension manager, config loader, and shell integration.

This GA release contains one fix on top of rc.8: the version info screen now displays
the installed extensions in a NAME/VERSION/PRIORITY/STATUS table (matching
`oradba_extension.sh list`) before the integrity check. No API changes.

---

## What Changed Since rc.8

### Fixed - compact display in `oradba_version.sh -i`

- **`src/bin/oradba_version.sh`** `show_installed_extensions()`: compact bullet
  format rendered before the integrity check:

  ```text
  Extensions (1):
    - odb_datasafe   1.0.9      [enabled] (50)
  ```

  Replaces the old table format (header/separator/Total rows). Colour-coded:
  `[enabled]` in green, `[disabled]` in yellow.

- **`src/bin/oradba_version.sh`** `version_info()`: compact `Installation:` block:

  ```text
  Installation:
    Installed:    2026-07-09 10:39 UTC
    Version:      1.0.0
    Method:       update
    User:         stefan.oehrli
    Mode:         standalone
  ```

  Date reformatted from ISO-8601 to `YYYY-MM-DD HH:MM UTC`; label alignment via
  `printf "  %-14s%s\n"`.

- **`src/bin/oradba_install.sh`** `version_compare()`: strip `-dev` suffix before
  comparing so `1.0.0-dev` is not incorrectly treated as older than
  `1.0.0-rc.8-dev` (lexicographic `d` < `r` false-positive).

## What Is Included (rc.1 through rc.8 highlights)

| Area | Key Changes |
|------|-------------|
| Installer | `--update --github` safe self-guard; symlink preservation in `etc/`; SHA256 checksum; `--coexist` mode; basenv detection |
| Extension manager | `oradba_extension.sh add/update/list/info/validate`; priority ordering; enable/disable; per-extension checksums |
| Config loader | `oradba_core.conf` / `oradba_standard.conf` cascade; `ORADBA_PS1_DIRTRIM`; `ORADBA_COEXIST_MODE` |
| Version tool | `oradba_version.sh --info/--verify/--update-check`; extension table; additional-files warning |
| Shell safety | `set -euo pipefail` throughout; safe indirect expansion `${!var:-}`; `set -e`-safe exit-code capture |
| Testing | BATS suite covering installer lifecycle, config loader, extension discovery, version check |

---

## Upgrade from rc.x

```bash
oradba_install.sh --update --github
```

The installer detects the running rc version and upgrades in place.
Runtime-managed files (`etc/oradba_local.conf`, `etc/sid.*.conf`, symlinks) are
preserved across the update.

---

## Known Limitations

- Config loader (`oradba_apply_config_section`) requires `[SECTION]` headers in
  config files. Shipped configs use plain shell exports; the loader treats them as
  `[DEFAULT]`. Fix planned for v1.1.0 (issue #180).
