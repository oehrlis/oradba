# Release Notes v1.0.0-rc.1

<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-27
**Type:** Release Candidate (Stabilisation RC - M9)
**Soak window:** 2026-06-27 to 2026-07-10 (adjusted; owner on vacation from 2026-07-12)

## Overview

Version 1.0.0-rc.1 is the first release candidate for OraDBA v1.0.0. All
blocking Change-Freeze items (CF-001 to CF-010, CF-015, CF-017, CF-018,
CF-034) have been verified closed. The public API surface is frozen. This
release starts a soak window targeting 2026-07-10 before the final v1.0.0 tag.

No new features are included in this RC. The focus is stabilisation,
regression coverage, and readiness verification.

---

## What the 30-day Soak Validates

The RC soak confirms that:

1. No Critical or High regressions emerge in real-world Oracle DBA environments
2. The frozen public API (`oradba_build_environment`, `execute_plugin_function_v2`,
   `oradba_registry_get_*`, `oradba_registry_discover_all`, `ORADBA_BASE`,
   `ORADBA_LOAD_PDB_ALIASES`) is stable and backwards-compatible
3. Docker integration against Oracle AI Database 26ai completes without failures
4. `make test-full` and `make validate-docs-counts` remain green on main throughout

## How to Promote to v1.0.0

After the 30-day soak window (not before 2026-07-27):

1. Confirm no Critical/High regressions filed against v1.0.0-rc.1
2. Run Docker integration test and record results in `doc/v1.0.0-readiness.md`
3. Mark remaining open items `[x]` in `doc/v1.0.0-readiness.md`
4. Set `VERSION` to `1.0.0` and add `## [1.0.0]` section to `CHANGELOG.md`
5. Create `doc/releases/v1.0.0.md`
6. Tag: `git tag -a v1.0.0 HEAD -m "Release v1.0.0"`
7. Push tag and let the release workflow create the GitHub Release
