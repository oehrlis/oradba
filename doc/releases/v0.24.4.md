# Release Notes v0.24.4
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-05-02
**Type:** Patch Release (Performance + Bug Fix)

## Overview

Version 0.24.4 eliminates subshell anti-patterns across all library, script, and
test files, and fixes an `unbound variable` crash in `extensions.sh` introduced
when `set -u` is active.

On macOS each `$(command)` fork costs approximately 30 ms. Replacing
`echo "$var" | cut/sed/grep/tr` chains with bash parameter expansion and
`[[ =~ ]]` / `BASH_REMATCH` removes the overhead entirely - no child processes,
no IPC, portable to bash 3.2+.

---

## Fixed

### `src/lib/extensions.sh`: Unbound Variable Crash on First Source

`oradba_version.sh -i` aborted with:

```text
/opt/oracle/local/oradba/lib/extensions.sh: line 23: ORADBA_EXTENSIONS_LOADED: unbound variable
```

**Root cause:** The multiple-sourcing guard referenced `${ORADBA_EXTENSIONS_LOADED}` without
a `:-` default. When `set -u` (nounset) is active and the library is sourced for the first
time, the variable is unset and bash treats the bare reference as an error.

```bash
# Before (fails under set -u on first source):
[[ -n "${ORADBA_EXTENSIONS_LOADED}" ]] && return 0

# After (safe - evaluates to empty string when unset):
[[ -n "${ORADBA_EXTENSIONS_LOADED:-}" ]] && return 0
```

This is the only library file that was missing the `:-` default; all other library files
(`oradba_common.sh`, `oradba_env_parser.sh`, etc.) already use the correct pattern.

## Changed

### Performance: Subshell Anti-Patterns Eliminated - Source Files

All `$(echo "$var" | external-command)` chains in loops and hot paths replaced with
bash built-ins. Estimated savings per loop iteration on macOS: **~30 ms per eliminated fork**.

| File | Pattern replaced | Replacement |
|---|---|---|
| `src/lib/oradba_common.sh` (`oradba_log`) | `echo "$level" \| tr '[:lower:]' '[:upper:]'` | `case` lookup table |
| `src/lib/oradba_env_parser.sh` | `echo "$field" \| sed 's/^ *//'` (×7 per line) | `${var#"${var%%[![:space:]]*}"}` |
| `src/bin/oraup.sh` | `echo "$ep" \| grep PROTOCOL \| cut …` + `tr` (×4 per endpoint) | `[[ =~ ]] + BASH_REMATCH` |
| `src/bin/oradba_version.sh` | `echo "$line" \| sed` in checksum loop | bash pattern substitution |
| `src/lib/oradba_home_discovery.sh` | `echo "$name" \| tr '[:upper:]' '[:lower:]'` in discovery loop | `${var,,}` + `tr` fallback |
| `src/lib/plugins/database_plugin.sh` | `echo … \| grep -oP 'pmon_\K\w+'` | `[[ =~ ]] + BASH_REMATCH` |
| `src/lib/plugins/oud_plugin.sh` | `grep -oP` (PCRE) | `grep -E` + `BASH_REMATCH` |
| `src/lib/plugins/plugin_interface.sh` | `grep -oP` (PCRE) | `grep -E` + `BASH_REMATCH` |
| `src/lib/plugins/datasafe_plugin.sh` | `grep -oP` (PCRE) | `grep -o` + parameter expansion |
| 10+ further `src/lib/` files | `echo "$v" \| tr` for case normalization | `${v^^}`/`${v,,}` + `tr` fallback |

**macOS/BSD compatibility:** All `grep -oP` (PCRE) instances replaced with `grep -E` or
`[[ =~ ]]` + `BASH_REMATCH`. BSD grep on macOS does not support `-P`; these patterns were
silently producing wrong results on macOS.

**bash 3.2 portability:** Every `${var^^}`/`${var,,}` usage includes a
`2>/dev/null || $(printf '%s' "$var" | tr ...)` fallback, keeping compatibility with the
default bash on macOS (3.2.57).
