## What's New in v0.10.4

### Bug Fix: Help Command Argument Parsing üêõ

Fixed the `oradba help <topic>` command to properly handle arguments:

**Problem:**

- `oradba help aliases` was showing main help instead of alias help
- `oradba help scripts` was showing main help instead of scripts list
- Only direct format `oradba aliases` worked correctly

**Solution:**

- Now detects when 'help' is the first argument
- Shifts arguments to process the actual topic
- Both formats work correctly:
  - `oradba help aliases` [OK]
  - `oradba aliases` [OK]

**Examples:**

```bash
# All these now work correctly:
oradba help aliases      # Show alias reference
oradba help scripts      # List available scripts
oradba help variables    # Show environment variables
oradba help config       # Configuration system info
oradba help sql          # SQL script help
oradba help online       # Open docs in browser
```
