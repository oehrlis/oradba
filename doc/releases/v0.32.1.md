# Release v0.32.1

## Summary

Hotfix release addressing an unbound variable error introduced by the RF-07 strict
mode change in v0.32.0. The `build_installer.sh` script was updated with
`set -euo pipefail`, but the `-u` flag caused a CI failure when `ORADBA_BUILD_SUFFIX`
was not set in the environment.

## Changes

- Fixed `build_installer.sh` to use `${ORADBA_BUILD_SUFFIX:-}` with a safe default,
  preventing the unbound variable error under `set -euo pipefail`.
- Added explicit `TARBALL_URL=""` initialization to avoid a second unbound variable
  reference later in the same script.

## Upgrade Notes

No functional changes. This is a build-pipeline fix only.
