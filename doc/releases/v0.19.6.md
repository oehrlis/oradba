# Release Notes v0.19.6

**Release Date:** 2026-01-22  
**Type:** Minor Release (Debug Support Enhancement - Phase 2)

## Overview

Version 0.19.6 completes the comprehensive debug support initiative by adding 
Phase 2 debug capabilities to OraDBA's Oracle management tools. This release 
builds on the Phase 1 infrastructure debug support from v0.19.5, providing 
complete visibility into Oracle database and listener operations.

## What's New

### üîç Phase 2 Debug Support for Management Tools

**Enhanced Database Management** in oradba_dbctl.sh

- Added comprehensive debug logging for database lifecycle operations
- Debug messages track startup/shutdown sequences, timeout handling, and error recovery
- Detailed tracing of Oracle Home detection, SQL operations, and process monitoring
- Enable with `ORADBA_DEBUG=true` or `--debug` flag
- Example: `ORADBA_DEBUG=true oradba_dbctl.sh start ORCL` shows complete startup trace

**Listener Management** in oradba_lsnrctl.sh

- Added debug logging for listener operations across multiple Oracle Homes
- Debug messages track environment setup, port detection, and status verification
- Detailed visibility into TNS_ADMIN configuration and listener discovery
- Enable with `ORADBA_DEBUG=true` or `--debug` flag  
- Example: `oradba_lsnrctl.sh --debug status` shows listener detection process

**Service Orchestration** in oradba_services.sh

- Added debug logging for high-level service coordination
- Debug messages track startup/shutdown ordering, configuration loading, and dependency management
- Detailed tracing of service command construction and execution
- Enable with `ORADBA_DEBUG=true` or `--debug` flag
- Example: `oradba_services.sh --debug restart` shows complete orchestration flow

## Technical Details

### Debug Coverage Areas

**Database Operations (oradba_dbctl.sh)**
- Database startup sequencing and parameter validation
- SQL*Plus session management and query execution
- Process monitoring and timeout handling
- Error recovery and retry logic

**Listener Management (oradba_lsnrctl.sh)**
- Multi-home listener discovery and environment setup
- Port detection and status verification
- TNS_ADMIN configuration and file resolution
- Listener service state transitions

**Service Orchestration (oradba_services.sh)**
- Service startup/shutdown ordering configuration
- Command construction and execution flow
- Configuration file loading and parameter parsing
- Inter-service dependency management

### Usage Examples

```bash
# Individual script debug
ORADBA_DEBUG=true oradba_dbctl.sh start ORCL
oradba_lsnrctl.sh --debug status LISTENER
oradba_services.sh --debug restart

# Combined operations
ORADBA_DEBUG=true oradba_services.sh start  # Shows full orchestration
```
