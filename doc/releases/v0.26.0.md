# Release v0.26.0 - Credential and Temp-File Hardening

**Release date:** 2026-06-26
**Previous version:** 0.25.0

## Summary

This release closes the M2 security gaps: credential exposure on disk and in
process listings, temp-file TOCTOU, eval injection via oratab/homes fields, and
the missing installer integrity check. It also adds a bash 4+ startup guard and
Oracle CLI pre-flight checks to the scripts that rely on them, plus focused
security regression tests.

## Changes

### Security

- **DBCA response file (CF-002):** The response file - which contains the SYS and
  SYSTEM passwords - is now created inside a per-run `mktemp -d` directory
  (mode 700) with the file set to mode 600. A script-level `EXIT` trap removes
  the directory on every exit path, including failure. The previous behaviour
  (predictable `/tmp/dbca_<SID>_<PID>.rsp` written with the default umask and
  deliberately preserved on error) is gone.
- **Installer integrity (CF-006):** `oradba_install.sh` downloads the companion
  `.sha256` file and verifies the tarball before extraction, preferring `shasum`
  (macOS/BSD) and falling back to `sha256sum` (Linux). When no checksum tool is
  available the installer now fails closed instead of warning and continuing.
- **SEPS wallet password (CF-020):** `get_seps_pwd.sh` refuses to read the
  `.wallet_pwd` file unless it is mode 600 and owned by the current user
  (BSD/Linux stat fallback). The recovered database password is no longer logged
  in cleartext; only a "password recovered" status line is emitted. A comment
  documents that the base64 layer is obfuscation, not encryption.
- **RMAN catalog credentials (CF-021):** The catalog connect string is moved into
  the RMAN script body (`connect catalog ...`) instead of the command line, so it
  no longer appears in `ps` output. The debug log redacts the user/password and
  keeps only the host/service portion.
- **eval injection in oraenv (CF-022):** The eval-based array manipulation over
  oratab, registry, and discovery fields is replaced with bash 4.3 namerefs.
  Every SID and home name is validated against a strict allowlist
  (`^[A-Za-z0-9_.+-]+$`) before being added to an array; invalid entries are
  skipped with a warning.
- **Predictable temp paths (CF-023):** PID-based temp paths in `oradba_rman.sh`,
  `oradba_homes.sh`, and `oradba_extension.sh` are replaced with `mktemp`
  exclusive creation and trap-based cleanup.

### Added

- **Bash 4+ startup guard (CF-011):** Scripts that use `declare -A`,
  `mapfile`/`readarray`, or `${var,,}`/`${var^^}` now abort early with a clear
  message on bash 3.x (`oradba_validate.sh`, `oradba_version.sh`,
  `oradba_dsctl.sh`, `oradba_dbctl.sh`, `oraup.sh`, `oradba_lsnrctl.sh`,
  `get_seps_pwd.sh`, `oradba_setup.sh`). `oradba_check.sh` reports the same as a
  non-fatal finding.
- **Oracle CLI pre-flight checks (CF-013):** `oradba_dbctl.sh` (sqlplus),
  `oradba_rman.sh` (rman), and `oradba_lsnrctl.sh` (lsnrctl) verify the tool is
  available before the first call.
- **Security regression tests (CF-009):** eval breakout via crafted oratab entry,
  wallet permission enforcement, and DBCA temp-file path.

## Notes

- The `--sys-password` / `--system-password` DBCA flags are retained (DR-3); their
  handling is improved by the CF-002 temp-file hardening.
