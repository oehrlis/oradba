# Release Notes v0.24.10
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-25
**Type:** Patch Release (Bug Fix)

## Overview

Version 0.24.10 fixes a second `set -e` related regression in `oraup.sh`: stopped
Data Safe connectors were displayed with a blank STATUS column instead of "stopped".

---

## Fixed

### `oraup.sh`: Stopped Data Safe connectors show no status

**Symptom:** Running connectors correctly display "running", but stopped connectors
show an empty STATUS column:

```text
Data Safe Connectors
------------------------------------------------------------------------------------------
NAME                 PORT (tcp/tcps)  STATUS        DATASAFE_BASE_HOME
------------------------------------------------------------------------------------------
dscon1               1562             running       /appl/oracle/product/exacc-wob-vwg-ha1
dscon2               1562                           /appl/oracle/product/exacc-wob-vwg-ha2
```

**Root cause:** `oradba_get_product_status` communicates status via both stdout
and exit code:

- Exit code 0 + stdout "running" → connector is up
- Exit code 1 + stdout "stopped" → connector is down
- Exit code 2 + stdout "unavailable" → error

With `set -euo pipefail` active (line 16 of `oraup.sh`), the pipeline:

```bash
status=$(oradba_get_product_status ... | grep -v "^\[" | tr '[:upper:]' '[:lower:]')
```

uses `pipefail`, so the exit status of the assignment is the rightmost non-zero exit
code in the pipeline — in this case 1 from `oradba_get_product_status`. A simple
assignment `var=$(...)` with a non-zero result triggers `set -e`, killing the
background subshell before `echo "${status}|${port_display}" > "$temp_file"` is
reached. The temp file stays empty; the display loop prints a blank status cell.

**Fix:** Append `|| true` to the assignment:

```bash
status=$(oradba_get_product_status ... | grep -v "^\[" | tr '[:upper:]' '[:lower:]') || true
```

The `|| true` prevents `set -e` from triggering regardless of the pipeline exit
code. The stdout capture ("stopped") is preserved correctly in `$status`.

---

## Context

This is the third `set -e` / `pipefail` bug fixed in the `oraup.sh` Data Safe
display path across the v0.24.x series:

| Version | Fix |
|---------|-----|
| v0.24.9 | `((idx++))` at idx=0 exited the collection loop before any rows were displayed |
| v0.24.10 | `status=$(pipeline_exit_1)` exited the subshell before the temp file was written |

## Upgrade Notes

- Drop-in replacement for v0.24.9 — no config changes required
- After updating, stopped Data Safe connectors will show "stopped" in `oraup.sh`
