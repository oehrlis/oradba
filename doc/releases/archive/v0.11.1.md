# Release v0.11.1

**Release Date:** 2026-01-02  
**Release Type:** Patch Release

## Overview

This patch release fixes test compatibility issues and enhances visibility into
coexistence mode configuration. It includes workflow improvements to ensure
documentation deploys correctly on every release.

## What's Fixed

### Test Compatibility

Fixed 6 service management tests that were checking for the old `alias name=`
syntax pattern. After implementing the safe alias system in v0.11.0, all alias
definitions now use `safe_alias name 'command'` format.

**Tests Updated:**

- `aliases.sh contains dbctl alias`
- `aliases.sh contains dbstart alias`
- `aliases.sh contains dbstop alias`
- `aliases.sh contains lsnrctl alias`
- `aliases.sh contains orastart alias`
- `aliases.sh contains orastop alias`

**Impact:** All 492 tests now pass, including full CI/CD pipeline.

### Documentation Workflow

Added version tag trigger (`v*.*.*`) to the documentation deployment workflow.

**Problem:** Documentation workflow only triggered on `release: published` event,
which was unreliable depending on release timing.

**Solution:** Workflow now triggers directly on version tag push, ensuring
documentation is always deployed when a new version is released.

**Triggers:**

- Version tag push (primary)
- Release published (backup)
- Documentation file changes
- Manual workflow dispatch

## What's New

### Version Info Enhancement

The `oradba_version.sh -i` command now displays coexistence mode information
from the `.install_info` file.

**New Fields Displayed:**

```bash
$ oradba_version.sh -i

OraDBA Version Information
==========================
Version:       0.11.1
Install Path:  /opt/oracle/local/oradba

Installation Details:
  Installed:     2026-01-02T12:34:56Z
  Version:       0.11.1
  Method:        embedded
  User:          oracle
  Coexist Mode:  basenv        # NEW: Shows standalone or basenv
  BasEnv:        yes            # NEW: Shows yes or no
```

**Benefits:**

- Quick visibility into coexistence mode status
- Easy verification of BasEnv detection during installation
- Helpful for troubleshooting coexistence issues
- No need to manually check configuration files

## Installation

### New Installation

```bash
# Download installer
curl -LO https://github.com/oehrlis/oradba/releases/download/v0.11.1/oradba_install.sh
chmod +x oradba_install.sh

# Install
./oradba_install.sh --prefix /opt/oracle/local/oradba
```

### Update from v0.11.0

```bash
# Download installer
curl -LO https://github.com/oehrlis/oradba/releases/download/v0.11.1/oradba_install.sh
chmod +x oradba_install.sh

# Update existing installation
./oradba_install.sh --prefix /opt/oracle/local/oradba --update
```

No configuration changes required - update is seamless.
