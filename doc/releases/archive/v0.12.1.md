# OraDBA v0.12.1 Release Notes

**Release Date:** January 2, 2026  
**Type:** Patch Release  
**Focus:** Bug Fixes for v0.12.0

## Overview

Version 0.12.1 is a patch release that fixes two issues discovered immediately after the
v0.12.0 release. This release addresses documentation build failures and improves
consistency in command-line interface formatting.

## What's Fixed

### Documentation Build Fix

**Issue:** The mkdocs documentation build failed in strict mode due to broken relative
links in the extension system user guide.

**Resolution:** Changed relative paths to absolute GitHub URLs for cross-repository
references:

- Developer documentation: `../../doc/extension-system.md` →
  `https://github.com/oehrlis/oradba/blob/main/doc/extension-system.md`
- Example extension: `../../doc/examples/extensions/customer/` →
  `https://github.com/oehrlis/oradba/tree/main/doc/examples/extensions/customer`

**Impact:** Documentation now builds successfully and deploys to GitHub Pages.

**Affected Files:**

- `src/doc/18-extensions.md`

### Extension Script Usage Formatting

**Issue:** The `oradba_extension.sh` script displayed ANSI color codes literally in its
usage output (e.g., `\033[1mUSAGE\033[0m`) instead of rendering colors, creating visual
clutter and inconsistency with other OraDBA scripts.

**Resolution:** Removed color codes from usage/help output to match the standard format
used by other OraDBA management scripts (`oradba_services.sh`, `oradba_dbctl.sh`, etc.).
Colors are still used appropriately in command output (list, info, validate).

**Impact:** Cleaner, more professional help text that follows OraDBA conventions.

**Affected Files:**

- `src/bin/oradba_extension.sh`

## Upgrade Notes

### From v0.12.0

This is a **drop-in replacement** for v0.12.0. No configuration changes or migration
steps are required.

**Installation:**

```bash
# Download and install
curl -L -o oradba_install.sh \
  https://github.com/oehrlis/oradba/releases/download/v0.12.1/oradba_install.sh
chmod +x oradba_install.sh && ./oradba_install.sh
```

**Verification:**

```bash
# Check version
${ORADBA_PREFIX:-/opt/oradba}/bin/oradba_version.sh
# Should show: 0.12.1

# Test extension script help
${ORADBA_PREFIX:-/opt/oradba}/bin/oradba_extension.sh help
# Should display clean, formatted help text
```

### From Earlier Versions

If upgrading from v0.11.x or earlier, review the [v0.12.0 release notes](v0.12.0.md)
for the major extension system features and changes.
