## What's Changed

### Added

- Central logging support for 53 SQL scripts with SPOOL statements
- Dynamic log directory configuration via `$ORADBA_LOG`
- Automatic fallback to current directory when `$ORADBA_LOG` not set
- Portable error handling across different SQL execution environments

### Changed

- Removed `rman` alias to prevent overriding Oracle's native command
- Updated `rmanc`, `rmanh`, `rmanch` aliases for better flexibility
- Enhanced project validation checks for documentation and test files

### Fixed

- SQL script temp file naming to use `${USER}` instead of `$$` for cross-platform compatibility
- HOST command error handling for SQL Developer and Windows compatibility

## Installation

### Quick Install (Linux/Unix)

```bash
curl -L https://github.com/oehrlis/oradba/releases/download/v0.9.5/oradba_install.sh | bash
```

### Manual Installation

```bash
# Download tarball
wget https://github.com/oehrlis/oradba/releases/download/v0.9.5/oradba-0.9.5.tar.gz

# Extract and install
tar xzf oradba-0.9.5.tar.gz
cd oradba-0.9.5
./install.sh
```

### Using Centralized SQL Logging

```bash
# Set log directory (optional)
export ORADBA_LOG=/var/log/oracle

# Run any SQL script - logs automatically go to $ORADBA_LOG
sqlplus / as sysdba @aud_policies_show_aud.sql

# Log file created: /var/log/oracle/aud_policies_show_aud_orcl_20260101_143045.log
```
