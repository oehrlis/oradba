# Release v0.18.3 - File Header Standardization & Project Validation

**Release Date:** 2026-01-13  
**Type:** Maintenance Release (Standardization & Validation)

## Overview

Maintenance release focused on standardizing file headers across the entire
codebase and enhancing project structure validation. This release ensures
consistency across 150+ files and improves project integrity checks to match the
current repository structure.

## File Headers Standardization

### Comprehensive Header Updates

Updated all source file headers to version 0.18.3 and date 2026.01.13, ensuring
consistent formatting and metadata across the entire project.

**Files Updated:**

- **22 Shell Scripts** (`src/bin/`): Core executable scripts including
  `oraenv.sh`, `oradba_install.sh`, `oradba_homes.sh`, `oradba_extension.sh`,
  `oradba_dbctl.sh`, `oradba_lsnrctl.sh`, `oradba_services.sh`, `oradba_rman.sh`,
  `oradba_sqlnet.sh`, `dbstatus.sh`, `oraup.sh`, `sync_to_peers.sh`,
  `sync_from_peers.sh`, and more

- **10 Configuration Files** (`src/etc/`): Configuration files and rlwrap
  completion definitions including `oradba_core.conf`, `oradba_standard.conf`,
  `oradba_services.conf`, `oradba_homes.conf.template`, `sid._DEFAULT_.conf`,
  and rlwrap completion files

- **126 SQL Scripts** (`src/sql/`): All SQL utility scripts including `login.sql`,
  `db_info.sql`, audit scripts, security scripts, and TDE utilities

- **5 Build Scripts** (`scripts/`): Build and utility scripts including
  `build_pdf.sh`, `archive_github_releases.sh`, `validate_project.sh`

**Header Format:** Consistent structure across all file types (shell, SQL,
config, RMAN):

```bash
# Name........: filename.sh
# Author......: Stefan Oehrli (oes) stefan.oehrli@oradba.ch
# Editor......: Stefan Oehrli
# Date........: 2026.01.13
# Revision....: 0.18.3
# Purpose.....: Brief description of file purpose
# Notes.......: Additional notes or prerequisites
# Reference...: URL references if applicable
# License.....: Apache License Version 2.0
```

**Impact:** Improved code documentation, easier maintenance tracking, and
consistent metadata across entire codebase.

## Project Validation Enhancement

### Comprehensive Structure Validation

Significantly enhanced `scripts/validate_project.sh` to validate the complete
current repository structure with 80+ specific file checks.

#### Added Validations

**New Core Scripts:**

- `oradba_validate.sh`, `oradba_setup.sh`, `oradba_homes.sh`
- `oradba_extension.sh`, `oradba_dbctl.sh`, `oradba_lsnrctl.sh`

  etc.)

**Template Files:**

**New Test Files:**

- `test_oradba_homes.bats`, `test_extensions.bats`

- `test_oradba_sqlnet.bats`, `test_oradba_services.bats`

**Additional Scripts:**

- `scripts/select_tests.sh`, `scripts/build_pdf.sh`

**Updated Documentation Checks:**

- Removed numbered prefixes from documentation file checks
- Updated to new naming convention (e.g., `structure.md` instead of
  `03_structure.md`)

**New CI/CD Workflow:**

- Added validation for `docs.yml` workflow

#### Improved Permission Checks

Enhanced executable permission validation with loop-based checks for all scripts:

- Core scripts in `src/bin/`
- Build scripts in `scripts/`
- Library scripts in `src/lib/`

#### Removed Obsolete References

Cleaned up validation for removed/renamed files:

- Removed `backup_full.rman` check (obsolete template)
- Updated documentation naming checks
- Removed references to old numbered documentation files

**Impact:** Comprehensive project structure validation ensures integrity across
all components. Validation now passes with 0 errors and 0 warnings.
