# Release v0.32.2

## Summary

Second hotfix for the `build_installer.sh` strict-mode migration. The v0.32.1 fix
addressed the `ORADBA_BUILD_SUFFIX` unbound variable, but a second failure remained:
when the GitHub API returns a rate-limit response (HTTP 403), the JSON body contains
no `tag_name` field. The `grep` command in the extension-release parsing block exits 1
(no match), which kills the script under `set -o pipefail`.

## Changes

- Added `|| true` guards to the `grep` pipeline chains that parse `tag_name` and
  `browser_download_url` from the GitHub API response, preventing script exit on
  no-match when the API is rate-limited.

## Upgrade Notes

No functional changes. Build-pipeline fix only.
