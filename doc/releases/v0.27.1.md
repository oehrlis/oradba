# Release v0.27.1 - CI scope cleanup (CF-001 follow-up)

**Release date:** 2026-06-26
**Previous version:** 0.27.0

## Summary

Hotfix: `oradba_check.sh`, `oraup.sh`, and `oradba_setup.sh` were added to the CI
arithmetic-lint scope in M3 but were not cleaned of zero-start `(( var++ ))` patterns.
This release fixes the 8 occurrences and applies `shfmt` formatting to all four newly
scoped files so CI passes end-to-end.

## Changes

### Fixed

- `oradba_check.sh`: replaced 5 standalone `(( var++ ))` patterns
  (`CHECKS_PASSED`, `CHECKS_FAILED`, `CHECKS_WARNING`, `CHECKS_INFO`, `homes_found`)
  with `var=$(( var + 1 ))`; applied shfmt
- `oraup.sh`: replaced `((listener_count++))` with `listener_count=$(( listener_count + 1 ))`;
  applied shfmt
- `oradba_setup.sh`: replaced 2 `((ext_count++))` occurrences; applied shfmt
- `oradba_validate.sh`: applied shfmt (no arithmetic changes needed)

- `src/bin/oradba_check.sh`
- `src/bin/oraup.sh`
