# Release v0.27.0 - Path-Critical Test Coverage

**Release date:** 2026-06-26
**Previous version:** 0.26.0

## Summary

This release closes the M3 testing gaps. It adds behavioral coverage for the
path-critical environment libraries (validator, builder, home discovery, version
metadata, output formatting), smoke tests for five bin scripts that previously
had none, and per-script first-iteration regression tests that lock in the M1
from-zero arithmetic fix. The suite now carries a healthy share of explicit
error-path assertions. This milestone is tests-only - no `src/bin/` or `src/lib/`
source files were changed.

## Changes

### Added

- **Validator coverage (CF-008):** `tests/test_oradba_env_validator.bats` exercises
  `oradba_validate_oracle_home`, `oradba_validate_sid`, `oradba_check_oracle_binaries`,
  `oradba_check_db_running`, `oradba_get_db_status`, `oradba_get_db_version`, and
  `oradba_validate_environment` with happy-path and error-path cases using a mock
  ORACLE_HOME and fake binaries - no real Oracle install required.
- **Env-builder coverage (CF-008):** `tests/test_oradba_env_builder_unit.bats` is
  extended with behavioral tests for `oradba_clean_path`, `oradba_add_oracle_path`,
  `oradba_set_lib_path`, `oradba_detect_rooh` (including the orabasetab read-only
  flag), `oradba_is_asm_instance`, `oradba_set_oracle_vars`,
  `oradba_set_asm_environment`, `oradba_set_product_environment`, and an integration
  test for `oradba_build_environment` against a mock home tree.
- **Home-discovery coverage (CF-008):** `tests/test_oradba_home_discovery.bats`
  covers `detect_product_type`, `is_oracle_home`, `parse_oracle_home`,
  `list_oracle_homes` (filtering and ordering), and `is_subdirectory_of_oracle_home`
  using filesystem stubs and a temporary `oradba_homes.conf`.
- **Version-metadata coverage (CF-008):** `tests/test_oradba_version_metadata.bats`
  covers `get_oradba_version`, `version_meets_requirement`, and the
  `get_install_info` / `set_install_info` / `init_install_info` round-trip against
  an isolated `.install_info`.
- **Env-output smoke tests (CF-008):** `tests/test_oradba_env_output.bats` verifies
  clean sourcing, presence of all public helpers, and the basic output behaviour of
  the divider / key-value / home-section functions.
- **Admin-script smoke tests (F-014):** `tests/test_oradba_admin_scripts.bats` adds
  shebang, `bash -n` syntax, and help/run coverage for `oradba_logrotate.sh`,
  `sessionsql.sh`, `oradba_validate.sh`, `oradba_datasafe_debug.sh`, and
  `oradba_setup.sh`, plus functional checks for `oradba_logrotate.sh --list` and
  `oradba_validate.sh`.
- **First-iteration regression tests (CF-009 completion):** Per-script tests that
  drive a single iteration so counters increment from 0 and would fail if the M1
  from-zero arithmetic fix were reverted - `oradba_version_reports_single_home_correctly`
  (single-file `--verify`), `oradba_logrotate_install_single_config_succeeds`,
  `oradba_dbctl_first_iteration_does_not_abort`, `oradba_lsnrctl_first_iteration_does_not_abort`,
  `oradba_dbca_first_iteration_does_not_abort`, and `oradba_sqlnet_first_iteration_does_not_abort`.
- **Error-path ratio (F-013):** Explicit negative/error-path assertions across the
  new and existing files bring the error-path share above the 15% target.
- **Test map (Task 9):** `.testmap.yml` gains entries for `oradba_env_output.sh`
  and the five admin scripts, and the annotation reflects the new test/file counts.

## Notes

- Sourcing-only behaviour: `sessionsql.sh` forwards its argument to SQL*Plus and
  has no `--help`, so it is covered by shebang and syntax tests only to avoid
  launching an Oracle client during the run.
- `oradba_datasafe_debug.sh` has no `--help` handling and always emits its debug
  report; the smoke test asserts it runs to completion rather than checking a
  specific exit code.
- The M1+M2 CI guard steps (from-zero post-increment check, shfmt) were relabelled
  to M1+M2+M3 scope; because M3 changes no `.sh` files, the scanned file lists are
  unchanged.
