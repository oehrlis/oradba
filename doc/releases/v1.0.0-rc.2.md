## Overview

Version 1.0.0-rc.2 adds the BasEnv coexistence mode (Issue #93) that was
originally planned for v1.1.0 but is included here so it can be validated
as part of the v1.0.0 soak window. Stefan has a single production BasEnv

feature tested.

No regressions relative to v1.0.0-rc.1. All 1711 tests pass.

## What Changed Since rc.1

### Added - BasEnv Coexistence Mode (Issue #93)

- **`ORADBA_COEXIST_MODE`**: three-value mode selector (`standalone`, `basenv`,
  `basenv-maximal`). `standalone` is the unchanged default. `basenv` is
  auto-detected when BasEnv markers are present; it loads oradba libraries and
  `ORADBA_*` vars but leaves all Oracle vars, PATH, and SQLPATH to BasEnv.
  `basenv-maximal` is explicit opt-in and additionally loads non-conflicting
  aliases via `safe_alias()`.

- **`detect_basenv()`** in `src/lib/oradba_common.sh`: reusable library
  function that checks three markers in priority order: `BE_HOME` variable,
  `~/.BE_HOME` file, `~/.TVDPERL_HOME` file.

- **`src/etc/oradba_basenv.conf.example`**: ready-to-copy integration snippet
  for `${ETC_BASE}/basenv.conf` documenting both minimal and maximal mode.

- **`tests/test_basenv_coexist.bats`**: 11 Bats tests covering `detect_basenv()`,
  `safe_alias()` guard, SQLPATH deduplication, auto-detection, and the
  `_oradba_path_contains` guard path.

- **`doc/basenv_coexist.md`**: user-facing integration guide with detection,
  mode table, integration steps, protected vars/aliases, verification, and
  troubleshooting.

## Protected Variables (basenv mode)

In `basenv` and `basenv-maximal` mode oraenv.sh will NOT modify:

| Category | Variables |
|----------|-----------|
| Oracle core | `ORACLE_SID`, `ORACLE_HOME`, `ORACLE_BASE`, `ORACLE_DOC`, `ORACLE_TERM` |
| Oracle PDB | `ORACLE_PDB`, `ORACLE_PDB_SID` |
| Locale/net | `NLS_LANG`, `TNS_ADMIN`, `ORATAB` |
| Libs | `LD_LIBRARY_PATH` |
| BasEnv | All `BE_*` vars, `TVD_BASE`, `ETC_BASE`, `LOG_BASE` |
| Shell | `PS1`, `PS1BASH`, `PROMPT_COMMAND`, `PATH` (not prepended), `SQLPATH` (not rebuilt) |

## Integration

Add to `${ETC_BASE}/basenv.conf`:

```bash
# shellcheck source=/dev/null
[[ -f "${TVD_BASE}/oradba/bin/oraenv.sh" ]] && source "${TVD_BASE}/oradba/bin/oraenv.sh"
```

For maximal mode, add to `${ORADBA_BASE}/etc/oradba_local.conf`:

```bash
export ORADBA_COEXIST_MODE="basenv-maximal"
```

See `src/etc/oradba_basenv.conf.example` and `doc/basenv_coexist.md` for full details.

## How to Promote to v1.0.0

After the soak window (not before 2026-07-10):

1. Confirm no Critical/High regressions filed against v1.0.0-rc.2
2. Run Docker integration test and record results in `doc/v1.0.0-readiness.md`
3. Mark remaining open items `[x]` in `doc/v1.0.0-readiness.md`
4. Set `VERSION` to `1.0.0` and add `## [1.0.0]` date to `CHANGELOG.md`
5. Create `doc/releases/v1.0.0.md`
6. Check doc/review/risk-register.md and technical-debt-register.md — open items become GitHub issues, then delete doc/review/.
7. Tag: `git tag -a v1.0.0 HEAD -m "Release v1.0.0"`
8. Push tag and let the release workflow create the GitHub Release
