# v0.28.0 - Plugin Tiered Isolation (2026-06-26)

## Summary

Enforces the tiered plugin isolation model (DECISION 2, CF-004). All
state-changing plugin function calls from CLI scripts and libraries now route
exclusively through `execute_plugin_function_v2`. A dead no-op fallback branch
in `oradba_common.sh` has been removed (ARCH-011).

## Key Change: Tiered Isolation Model

State-changing functions (`plugin_detect_installation`, `plugin_check_status`,
`plugin_check_listener_status`) previously could be called directly after
sourcing a plugin file into the parent shell. This allowed plugin state
(`plugin_status`, `plugin_name`, environment variables) to leak into the
caller's shell context.

From v0.28.0, these functions are only reachable through
`execute_plugin_function_v2`, which runs the plugin in an isolated subshell.
The exception list for pure path-builders (`build_bin_path`, `build_lib_path`)
is formally documented in `doc/architecture.md` (Plugin Isolation Model).
