# Release v0.30.0 - Environment-Build Migration (M6)

**Release date:** 2026-06-27
**Previous version:** 0.29.0

## Summary

Milestone 6 completes the environment-build migration started in M5. The
`oraenv.sh` inline environment-building block is retired: `_oraenv_handle_oracle_home`
now delegates to `oradba_build_environment` from `oradba_env_builder.sh`,
which is the documented single public path for Oracle environment construction.

A single canonical DB open-mode function with a documented exit-code contract
replaces three divergent implementations. The vocabulary {OPEN, MOUNTED, NOMOUNT,
STARTED, SHUTDOWN} is now enforced consistently across `oradba_env_status.sh`,
`oradba_env_validator.sh`, and `oradba_db_functions.sh`.

New BATS tests in `test_oradba_env_migration.bats` verify environment parity and
canonical DB open-mode behaviour without requiring a live Oracle installation.

## Changes

### Changed

- **CF-017** - `_oraenv_handle_oracle_home` in `oraenv.sh` delegates to
  `oradba_build_environment`; `set_oracle_home_environment` is retained as a
  compatibility shim in `oradba_home_discovery.sh` but is no longer called
  directly from `oraenv.sh`.

- **CF-018** - `oradba_check_db_status` is the canonical DB open-mode function.
  Vocabulary fixed to {OPEN, MOUNTED, NOMOUNT, STARTED, SHUTDOWN} with documented
  exit-code contract (0 = accessible, 1 = SHUTDOWN). `oradba_get_db_status` in
  `oradba_env_validator.sh` is now a thin shim. `get_database_open_mode` in
  `oradba_db_functions.sh` normalises raw sqlplus output to canonical vocabulary.

### Added

- `tests/test_oradba_env_migration.bats` - Environment parity and DB open-mode
  tests covering CF-017 and CF-018 behaviour without a live Oracle install.
