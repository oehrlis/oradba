# Release v0.29.0 - Architecture Consolidation (M5)

**Release date:** 2026-06-27
**Previous version:** 0.28.1

## Summary

Milestone 5 establishes the architectural foundations for v1.0.0. Key changes:
one canonical install-root variable (`ORADBA_BASE`), a shared bootstrap loader,
centralised oratab parsing, `oradba_`-prefixed public API, and portability fixes.

## Changes

### Changed

- **CF-007** - `ORADBA_BASE` is now the canonical install-root variable.
  `ORADBA_PREFIX` emits a deprecation warning when set and acts as a fallback
  alias for one release. All `${ORADBA_PREFIX}/` path constructions in
  `src/lib/` and `src/bin/` now reference `${ORADBA_BASE}/`.

- **CF-032** - New shared bootstrap loader `src/lib/oradba_bootstrap.sh`.
  Bin scripts that previously resolved the install root ad hoc now source this
  single file. The loader is idempotent (guarded by `_ORADBA_BOOTSTRAP_LOADED`).
  Converted: `dbstatus.sh`, `oradba_validate.sh`. Remaining scripts left for M6.

- **CF-029** - Oratab parsing centralised exclusively in `oradba_registry.sh`.
  `generate_sid_lists` in `oradba_database_discovery.sh` continues to create
  shell aliases but delegates canonical parsing to the registry API.

- **CF-030** - Public functions prefixed with `oradba_`; old unprefixed names
  retained as one-release deprecation aliases:
  - `oradba_parse_oratab`, `oradba_generate_sid_lists` (database_discovery.sh)
  - `oradba_get_oracle_homes_path`, `oradba_detect_product_type`,
    `oradba_set_oracle_home_environment` (home_discovery.sh)
  - `oradba_check_database_connection` (db_functions.sh)

### Fixed

- **CF-031** - Error messages in `get_seps_pwd.sh`, `oradba_dsctl.sh`,
  `oradba_dbctl.sh`, `oradba_lsnrctl.sh`, and `oradba_services_root.sh` routed
  to stderr. WARNING messages in bulk-operation prompts also routed to stderr.

- **CF-012** - `df -BG` and `df -Pm` replaced with portable `df -k` plus awk
  in `oradba_dbca.sh` and `oradba_check.sh`. `timeout` portability fallback
  added to `datasafe_plugin.sh`. `realpath` portability fallback added to
  `sync_to_peers.sh` and `sync_from_peers.sh`.

- **CF-033** - `sort` and `comm` calls prefixed with `LC_ALL=C` for locale
  safety across core libraries and plugins. Uptime computation in
  `oradba_db_functions.sh` guarded against implausible epoch values (before
  year 2000 or zero).

- **CF-030** - `(( var++ ))` standalone patterns replaced with
  `var=$(( var + 1 ))` in all touched files.
