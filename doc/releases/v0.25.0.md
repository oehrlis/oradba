# Release v0.25.0 - Immediate Safety Net

**Release date:** 2026-06-26
**Previous version:** 0.24.11

## Summary

This release eliminates the recurring zero-start arithmetic defect class, adds
full strict mode to the two scripts that were missing it, fixes the plugin
interface template, and installs the preventive controls (regression tests, CI
lint guards, release pipeline integrity) that protect against recurrence.

## Changes

### Fixed

- Zero-start `(( counter++ ))` pattern replaced with `var=$(( var + 1 ))` in
  `oradba_dsctl.sh`, `oradba_dbctl.sh`, `oradba_lsnrctl.sh`, `oradba_dbca.sh`,
  `oradba_version.sh`, `oradba_logrotate.sh`, `oradba_sqlnet.sh`,
  `oradba_env_changes.sh`, `oradba_homes.sh`, and `oradba_extension.sh`
- Added `set -euo pipefail` to `oradba_homes.sh` (was missing entirely) and
  completed it in `oradba_extension.sh` (had only `set -o pipefail`)
- Removed duplicate `plugin_check_listener_status()` definition from
  `plugin_interface.sh`; one canonical definition now exists
- Release pipeline VERSION-vs-tag assertion: build now fails if VERSION file
  does not match the pushed git tag
- `make test-full` now correctly distinguishes bats failures from skipped tests

### Added

- 12 named regression tests for b76fe9c, 5e89542, 4db7ccf, cbcb942, bbf2540,
  and fa36489 defects
- CI lint step: standalone `(( var++ ))` patterns cause CI failure
- CI format check: `shfmt -d` format verification
