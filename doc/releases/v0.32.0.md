# Release Notes v0.32.0
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-27
**Type:** Minor Release (Release Engineering and Documentation - M8)

## Overview

Version 0.32.0 closes the release-gate and documentation-accuracy gaps identified
in the OraDBA roadmap milestone M8. The release workflow now enforces that a
`doc/releases/v<TAG>.md` file exists before a GitHub release can be created.
Registry API documentation has been corrected to match the actual pipe-delimited
8-field schema. A new `make validate-docs-counts` target keeps `.testmap.yml`
annotations in sync with the actual BATS file count.

A pre-push git hook (`make setup-dev`) and an explicit v1.0.0 readiness checklist
(`doc/v1.0.0-readiness.md`) are also included.

---

## Changed

- Release workflow fails with `exit 1` when release notes file is missing (CF-026)
- Registry API documented in `doc/api.md`: pipe delimiter (`|`), 8-field schema,
  4 real functions listed (CF-024)
- `make validate-docs-counts` added; `.testmap.yml` count annotation auto-verified
  against actual BATS file count (CF-025)
- `build_installer.sh` strict mode: `set -euo pipefail` (RF-07)

## Added

- `.githooks/pre-push` runs `make lint`; `make setup-dev` wires the hook (RF-13)
- `doc/v1.0.0-readiness.md` with explicit v1.0.0 release criteria (RF-14)
- `doc/releases/v0.24.5.md` retroactive release note (CF-026)

## Upgrade Notes

- Run `make setup-dev` once per clone to wire the pre-push hook
- No API or behavioral changes for end users
