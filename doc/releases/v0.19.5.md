# Release Notes v0.19.5

**Release Date:** 2026-01-22  
**Type:** Minor Release (Debug Support Enhancement)

## Overview

Version 0.19.5 introduces Phase 1 debug support for OraDBA's core infrastructure,
making troubleshooting and development significantly easier. This release adds
comprehensive debug logging to critical CLI tools and improves the overall
developer experience.

## What's New

### üîç Enhanced Debug Support

**Debug Logging** in oraup.sh

- Added comprehensive debug logging throughout the main OraDBA status tool
- Debug messages track library/plugin loading, oratab detection, installation classification, and display decisions
- Enable with `ORADBA_DEBUG=true` or `ORADBA_LOG_LEVEL=DEBUG`
- Example: `ORADBA_DEBUG=true oraup.sh` shows detailed execution trace
- Helps diagnose Oracle Home detection issues and plugin behavior

**Phase 1** Debug Flags in CLI Tools

- Added `--debug` flag and `ORADBA_DEBUG` environment variable support to:
  - `oradba_check.sh`: System prerequisites and Oracle environment verification
  - `oradba_extension.sh`: Extension management and discovery
- Instrumented decision points for better visibility into tool operations
- Backward compatible: no output changes unless debug enabled
- Usage examples:
  - `ORADBA_DEBUG=true oradba_check.sh --verbose`
  - `oradba_extension.sh --debug list`

## Bug Fixes

### üîß Plugin Loader Variable Assignment

**Fixed Critical** Variable Collision Bug

- Resolved `oradba_apply_oracle_plugin` function failing to assign plugin results to variables
- Root cause: Internal parameter name `result_var` collided when callers used the same variable name
- Solution: Changed internal parameter from `result_var` to `result_var_name`
- Affected DataSafe status detection in `oraup.sh` and `oradba_env_status.sh`
- Added regression test to prevent future variable name collisions

## Technical Details

### Debug Implementation Patterns

This release establishes consistent debug patterns across OraDBA:

- **Environment Variable**: `ORADBA_DEBUG=true` enables debug mode globally
- **CLI Flags**: `--debug` flag enables debug mode for individual script runs
- **Log Level**: `ORADBA_LOG_LEVEL=DEBUG` controls logging verbosity
- **Inline Functions**: Standalone scripts use `log_debug()` inline functions
- **Library Functions**: Scripts sourcing `oradba_common.sh` use `oradba_log DEBUG`
