# Release v0.10.5

**Release Date:** 2026-01-02  
**Release Type:** Minor Release

## Overview

This release adds user-mode (non-root) operation for log rotation, completes
naming conventions documentation, and includes several important fixes and improvements.

## New Features

### Log Rotation User Mode

Non-root operation support for `oradba_logrotate.sh` allows DBAs to manage log rotation without requiring root privileges.

**New Options:**

- `--install-user`: Set up user-specific logrotate configurations
- `--run-user`: Run logrotate as non-root user
- `--cron`: Generate crontab entry for automated rotation

**Benefits:**

- No root access required for log rotation
- User-specific state tracking in `~/.oradba/logrotate/state/`
- Flexible scheduling via user's crontab
- Isolated from system-wide logrotate

**Documentation:** See [15-log-management.md](../../src/doc/15-log-management.md) for complete guide.

### Naming Conventions Documentation

Comprehensive documentation of OraDBA naming conventions added to [development.md](../development.md).

**Covers:**

- Bash script patterns: `oradba_*`, `*_jobs.sh`, utility scripts
- Configuration files: `oradba_*.conf`, `sid.*.conf`
- Library, template, test, and build script conventions
- SQL script naming patterns

**Resolves:** Issue #4

## Changes

### SQL*Net Authentication Update

Changed authentication service from NTS to BEQ for Unix/Linux systems.

**Rationale:**

- NTS (Native Network Services) is Windows-specific
- BEQ (Bequeath) is the correct method for Unix/Linux OS authentication
- Updated templates: `sqlnet.ora.basic`, `sqlnet.ora.secure`

### Listener Alias Renamed

Renamed `lsnrctl` alias to `listener` to avoid conflict with Oracle's native command.

**Changes:**

- New alias: `listener` → calls `oradba_lsnrctl.sh` wrapper
- Preserved: `lsnr`, `lsnrh` → Oracle's native `lsnrctl`
- Unchanged: `lsnrstart`, `lsnrstop`, `lsnrrestart`, `lsnrstatus` → OraDBA wrappers

**Benefit:** Oracle's native `lsnrctl` command is no longer shadowed by alias.

## Installation

### New Installation

```bash
# Download installer
curl -LO https://github.com/oehrlis/oradba/releases/download/v0.10.5/oradba_install.sh

# Run installer
bash oradba_install.sh

# Or with custom prefix
bash oradba_install.sh --prefix /opt/oracle/local
```

### Upgrade from v0.10.4

```bash
# Re-run installer in same location
bash oradba_install.sh --prefix /opt/oracle/local

# Re-source environment
source /opt/oracle/local/oradba/bin/oraenv.sh
```

## Upgrade Notes

### For Existing Users

1. **Listener Alias Change:**
   - If you use `lsnrctl` alias, replace with `listener`
   - Or use Oracle's native `lsnrctl` directly

2. **SQL*Net Authentication:**
   - BEQ is now default for Unix/Linux
   - Existing NTS configurations will continue to work but are not recommended

3. **Log Rotation:**
   - System-wide (root) mode unchanged
   - New user-mode available for non-root scenarios

- All features enabled by default
- Follow standard installation guide: [02-installation.md](../../src/doc/02-installation.md)
