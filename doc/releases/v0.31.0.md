# Release v0.31.0 - Performance (M7)

**Release date:** 2026-06-27
**Previous version:** 0.30.0

## Summary

Milestone 7 cuts per-env-switch cost via three targeted improvements:
lazy library loading (CF-014), a PDB-alias gate (CF-015), and
hot-path fork reduction (CF-016).

## Changes

### Changed

- **CF-014** - `oradba_env_parser.sh`, `oradba_env_builder.sh`,
  `oradba_env_validator.sh`, and `oradba_env_config.sh` are now sourced
  lazily inside the first function that needs each library. A helper
  `_oraenv_ensure_lib` guards against double-sourcing. The duplicate
  `load_config_file` calls at the top of `oraenv.sh` are consolidated.

- **CF-015** - `generate_pdb_aliases` is gated by `ORADBA_LOAD_PDB_ALIASES`
  (default `false`). A per-SID session guard (`ORADBA_PDB_ALIASES_DONE_<SID>`)
  prevents repeated SQL*Plus calls when the same SID is switched to more
  than once in a shell session. The `--fast-silent` flag in `oraenv.sh`
  explicitly sets `ORADBA_LOAD_PDB_ALIASES=false`.

### Fixed

- **CF-016** - Hot-path fork reduction in `oradba_common.sh` and
  `oradba_home_discovery.sh`: `awk '{print $N}'` subshells on
  `parse_oracle_home` output replaced with bash parameter expansion;
  SQL noise filtering in `execute_db_query` replaced with a
  `while IFS= read` loop.
