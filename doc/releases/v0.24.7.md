# Release Notes v0.24.7
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-25
**Type:** Patch Release (Feature Addition)

## Overview

Version 0.24.7 adds a suite of SQL scripts (`odb_svc`) for managing role-aware
PDB services and a startup trigger in Oracle Data Guard environments. The trigger
`odb_pdb_service_trigger` automatically starts the correct `_RW` or `_RO` service
on PDB startup based on the current database role (PRIMARY / PHYSICAL STANDBY).

Service names are derived dynamically from the PDB name and `db_domain`:

- `<CON_NAME>_RW[.<db_domain>]` - active on PRIMARY
- `<CON_NAME>_RO[.<db_domain>]` - active on PHYSICAL STANDBY

## Added

### `odb_svc` - PDB Service Trigger Scripts for Data Guard

Seven SQL scripts covering the full lifecycle of role-aware PDB services:

| Script | Purpose |
|--------|---------|
| `odb_svc_service_trigger_create.sql` | Initial setup: creates services and trigger; aborts if objects already exist |
| `odb_svc_service_trigger_recreate.sql` | Idempotent re-creation: drops and recreates services and trigger; recommended for migration runbooks |
| `odb_svc_service_trigger_drop.sql` | Cleanup: stops and removes services, drops the trigger; silent when objects do not exist |
| `odb_svc_service_trigger_show.sql` | Read-only status: PDB identity/role, registered services, active services, trigger status and source |
| `odb_svc_service_trigger_manage.sql` | Manual control: `START` / `STOP` / `STATUS` actions; service names resolved via `LIKE '<PDB>\_R%'` |
| `odb_svc_service_trigger_check.sql` | Migration check: compares legacy `SERVICE_TRIGGER` with `odb_pdb_service_trigger`; outputs verdict |
| `odb_svc_service_trigger_check_all.sql` | CDB-level overview: trigger and service state for all open PDBs from `CDB$ROOT` |

**Usage - initial installation:**

```sql
CONNECT / AS SYSDBA
ALTER SESSION SET CONTAINER = MYPDB;
@odb_svc_service_trigger_create.sql
@odb_svc_service_trigger_show.sql
```

**Usage - re-migration / runbook:**

```sql
CONNECT / AS SYSDBA
ALTER SESSION SET CONTAINER = MYPDB;
@odb_svc_service_trigger_recreate.sql
@odb_svc_service_trigger_show.sql
```

**Requirements:**

- All scripts must be executed as SYSDBA after `ALTER SESSION SET CONTAINER = <pdb>`
- Exception: `odb_svc_service_trigger_check_all.sql` runs in `CDB$ROOT`
- Container context is enforced via `WHENEVER SQLERROR EXIT SQL.SQLCODE` and
  `RAISE_APPLICATION_ERROR`; incorrect context aborts SQL*Plus cleanly

**Trigger behaviour:**

The trigger fires `AFTER STARTUP ON DATABASE` inside the PDB and starts the
role-appropriate service while explicitly stopping the other. `START_SERVICE`
failures are logged to the alert log via `DBMS_SYSTEM.KSDWRT` and re-raised,
preventing the PDB from opening when a service cannot be started.
