# Release Notes v0.24.9
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-25
**Type:** Patch Release (Bug Fix)

## Overview

Version 0.24.9 fixes a regression in `oraup.sh` where Data Safe connectors were
registered and detected correctly but never displayed. The Data Safe section header
appeared in the output while the connector rows remained blank.

---

## Fixed

### `oraup.sh`: Data Safe connectors not displayed under `set -e`

**Symptom:** `oraup.sh` shows the Data Safe Connectors section header and column
labels but lists no connectors, even though `oradba_homes.sh list` shows all
`dscon*` entries as available:

```text
Data Safe Connectors
------------------------------------------------------------------------------------------
NAME                 PORT (tcp/tcps)  STATUS        DATASAFE_BASE_HOME
------------------------------------------------------------------------------------------

```

**Root cause:** The parallel status-check loop contained `((idx++))` as a leftover
counter increment. Under `set -euo pipefail` (mandatory for all OraDBA scripts),
an arithmetic expression that evaluates to `0` is treated as a failure and exits
the script immediately. On the first loop iteration `idx` is `0`, so `((idx++))` —
post-increment: returns the old value `0` — is falsy, triggering `set -e` exit.

The script exited after collecting the first connector's data into the arrays but
before the display loop at line 631 was reached. Because the section header is
printed before the collection loop, the blank-section output was the result.

**Fix:** Removed the `((idx++))` line. The counter was entirely unused:

- The collection arrays (`ds_names`, `ds_homes`, etc.) use `+=` to append.
- The display loop uses `for idx in "${!ds_names[@]}"` which reinitialises `idx`
  from the array indices — the value carried from the collection phase is irrelevant.

**Same root cause** as the `(( count++ ))` fix applied to the bats test suite in
v0.24.5 (`(( ++count ))` / `(( count += 1 ))` as safe alternatives when a counter
is actually needed).

---

## Affected Versions

v0.24.7 and v0.24.8 — both contain the `((idx++))` line. Earlier versions used a
different code path for the Data Safe display loop and were unaffected.

---

## Upgrade Notes

- Drop-in replacement for v0.24.8 — no config changes required
- `oradba_homes.conf` entries are not modified
- After updating, `oraup.sh` will show all `dscon*` connectors with status and port
