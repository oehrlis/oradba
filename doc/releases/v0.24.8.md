# Release Notes v0.24.8
<!-- markdownlint-disable MD013 MD060 -->

**Release Date:** 2026-06-25
**Type:** Patch Release (Bug Fix)

## Overview

Version 0.24.8 fixes a regression in `oradba_home_discovery.sh` that caused the
DESCRIPTION column to appear blank for all homes registered with an empty
`ALIAS_NAME` field (double-colon `::` in the config). The most visible symptom
was that all `dscon*` Data Safe connector entries showed no description in
`oradba_homes.sh list`.

---

## Fixed

### `oradba_home_discovery.sh`: Description wiped when `ALIAS_NAME` is empty

**Symptom:** `oradba_homes.sh list` shows an empty DESCRIPTION column for Data Safe
connector homes (`dscon1`–`dscon7`) even though the descriptions are present in
`oradba_homes.conf`:

```text
NAME            TYPE         STATUS       DESCRIPTION
dscon1          datasafe     available
dscon2          datasafe     available
```

**Expected output:**

```text
NAME            TYPE         STATUS       DESCRIPTION
dscon1          datasafe     available    Auto-discovered datasafe (oci=ds-conn-exacc-wob-vwg-ha1)
dscon2          datasafe     available    Auto-discovered datasafe (oci=ds-conn-exacc-wob-vwg-ha2)
```

**Root cause:** Both `parse_oracle_home()` and `list_oracle_homes()` contained a
backward-compatibility branch for old-format config files that did not have an
`ALIAS_NAME` field. In those files, position 5 (the alias slot) holds a description
string with spaces. The detection condition was:

```bash
if [[ -z "${h_alias}" ]] || [[ "${h_alias}" =~ [[:space:]] ]]; then
    # shift: treat h_alias as description, move h_desc → h_version
    h_version="${h_desc}"
    h_desc="${h_alias}"
    h_alias="${h_name}"
fi
```

The `[[ -z "${h_alias}" ]]` branch is wrong for new-format configs. When
auto-discovery writes `dscon1:...:datasafe:10::Auto-discovered datasafe (...):AUTO`,
the `IFS=:` split produces `h_alias=""` and `h_desc="Auto-discovered datasafe (...)"`
— both correct. But the empty-alias check fires the shift, moving the description
into `h_version` and leaving `h_desc` empty.

**Fix:** Split the condition into two explicit branches:

```bash
if [[ "${h_alias}" =~ [[:space:]] ]]; then
    # Old-format config: position 5 is a description (contains spaces) — shift
    h_version="${h_desc}"
    h_desc="${h_alias}"
    h_alias="${h_name}"
elif [[ -z "${h_alias}" ]]; then
    # New-format config: empty alias means "use home name" — no shift
    h_alias="${h_name}"
fi
```

Fix applied in two locations: `parse_oracle_home()` (line 219) and
`list_oracle_homes()` (line 265) in `src/lib/oradba_home_discovery.sh`.

---

## Affected Configurations

- Homes auto-discovered with an empty `ALIAS_NAME` field, i.e. config lines of the
  form `NAME:PATH:TYPE:ORDER::DESCRIPTION:VERSION` (double-colon before description)
- Manually added homes with explicit alias are unaffected
- Old-format configs without the alias field (description in position 5) continue
  to work correctly via the spaces-in-alias branch

---

## Upgrade Notes

- Drop-in replacement for v0.24.7 — no config changes required
- Existing `oradba_homes.conf` entries are not modified by the fix
- After updating, `oradba_homes.sh list` will show descriptions immediately
