# v1.0.0 Readiness Checklist

<!-- markdownlint-disable MD013 -->

This document defines the explicit criteria that must all be satisfied before
the `v1.0.0` tag is created. Each item maps to a Change-Freeze (CF) or
Refactoring (RF) item from the OraDBA roadmap.

**Owner:** Stefan Oehrli
**Target:** v1.0.0 release

---

## Blocking Change-Freeze Items

All of the following CFs must be closed (resolved and merged to `main`):

- [ ] CF-001 - Core library `set -euo pipefail` audit complete
- [ ] CF-002 - Registry API field-separator standardised (pipe `|`)
- [ ] CF-003 - Plugin interface 8-function contract enforced
- [ ] CF-004 - Environment builder single public entry-point (`oradba_build_environment`)
- [ ] CF-005 - oratab parsing exclusively in `oradba_registry.sh`
- [ ] CF-006 - Logging wrapper consistent across all scripts
- [ ] CF-007 - `ORADBA_BASE` canonical; `ORADBA_PREFIX` deprecation warning live
- [ ] CF-008 - Bootstrap loader (`oradba_bootstrap.sh`) wired to all bin scripts
- [ ] CF-009 - `shfmt -i 4 -bn -ci -sr` applied to all `.sh` files, CI enforces
- [ ] CF-010 - `shellcheck` passes with zero warnings on all `.sh` and `.bats` files
- [ ] CF-015 - PDB alias generation gated by `ORADBA_LOAD_PDB_ALIASES`
- [ ] CF-017 - `oraenv.sh` delegates environment construction to `oradba_build_environment`
- [ ] CF-018 - Single canonical DB open-mode function with documented exit-code contract
- [ ] CF-019 - Hot-path fork reduction: deduplication, timestamp gating, cached checks
- [ ] CF-024 - Registry API documented: pipe delimiter, 8-field schema, 4 real functions
- [ ] CF-034 - All scripts have `--dry-run`, `--yes`, `--help` flags

---

## Quality Gate Checks

All of these must pass on the final commit before tagging:

- [ ] `make lint` exits 0 (no shellcheck warnings, no markdownlint errors, shfmt clean)
- [ ] `make test-full` passes with 0 failures (skips are acceptable)
- [ ] `make validate-docs-counts` exits 0 (`.testmap.yml` count matches actual)
- [ ] `awk '{if(length>120) print NR": "length" chars"}' CHANGELOG.md` - empty output

---

## Version and Release Artifacts

- [ ] `VERSION` file contains exactly `1.0.0`
- [ ] `CHANGELOG.md` has a `## [1.0.0] - YYYY-MM-DD` section with all changes
- [ ] `doc/releases/v1.0.0.md` exists and documents the release

---

## Integration and Validation

- [ ] Docker integration run executed against Oracle AI Database 26ai
- [ ] Results recorded (pass/fail counts, environment, date)
- [ ] No Critical or High regressions found during the 30-day RC soak period
- [ ] RC soak start date recorded and 30 days elapsed since then

---

## API and Compatibility

- [ ] Deprecation warnings present and tested for `ORADBA_PREFIX`
- [ ] Public API surface declared frozen in `doc/api.md` (marked as stable)
- [ ] All public functions have `oradba_` prefix (old names kept as deprecation aliases)
- [ ] No undocumented breaking changes since the last release

---

## How to Use This Document

1. Work through all items top-to-bottom; mark each `[x]` when resolved
2. For blocking CFs: verify the fix is merged to `main` and CI is green
3. For quality gates: run the commands and paste output into a comment
4. Only create the `v1.0.0` tag after ALL items are checked
