# Log Files

This directory contains log files generated by OraDBA scripts when not using a
centralized log directory.

## Overview

OraDBA scripts generate log files for tracking operations, debugging, and auditing.
By default, logs are stored based on the `ORADBA_LOG` environment variable.

## Log Location Priority

1. **`$ORADBA_LOG`** - Centralized log directory (recommended for production)
2. **`$ORADBA_BASE/log`** - Installation log directory (fallback)
3. **Current directory** - Last resort if above are not writable

## Log File Naming

Log files follow the format: `<script>_<sid>_<timestamp>.log`

- `script`: Name of the executing script
- `sid`: Database SID (lowercase)
- `timestamp`: YYYYMMDD_HH24MISS format

Examples:

```text
oradba_rman_proddb_20260115_143022.log
oraenv_testdb_20260115_090512.log
```

## Configuration

Set centralized logging in your environment:

```bash
# In ~/.bashrc or oradba_customer.conf
export ORADBA_LOG=/var/log/oracle
```

## Log Rotation

See [templates/logrotate/](../templates/logrotate/README.md) for logrotate
configuration templates to manage OraDBA log files.

## Cleanup

Manual cleanup:

```bash
# Remove logs older than 30 days
find $ORADBA_LOG -name "*.log" -mtime +30 -delete
```
